Task 1 (גרסה 0.0.1 – הושלם):
התקנת פרויקט נקי

**הגדרת שלב:**

- יצירת פרויקט Next.js 15 חדש, נקי, המוכן לעבודה עם:
  - App Router.
  - TypeScript.
  - Tailwind CSS.
  - pnpm בתור מנהל חבילות.
- חיבור מבנה תיקיות בסיסי כפי שהוגדר תאורטית:
  - שכבת `app`.
  - שכבת `src/core`.
  - שכבת `src/modules`.
- וידוא שהפרויקט רץ מקומית ושמחובר ל־Repo ול־Vercel.

**הערות:**

- בשלב זה עדיין לא משתמשים ב־Supabase בפועל, אלא רק דואגים שמבנה הפרויקט מוכן.
- כל פקודות ההתקנה, שמות קבצים, ותוכן הקבצים הראשון – ייכתבו בחלק הזה באופן מפורט וטכני (מחוץ לחלק התאורטי).

**סטטוס:** גרסה 0.0.1 נסגרה בהצלחה.

Task 2:
יצירת ה־Layout הראשי

**הגדרת שלב:**

- בניית Layout ראשי בגישת Mobile First עם Header ו-Footer קבועים שאינם נעלמים בגלילה.
- ב־Header יוצג לוגו ממורכז.
- ב־Footer יוצג לוגו ממורכז, לצד הטקסט `© 2025 All rights reserved`.
- שימוש בפלטת צבעים יוקרתית של שחור מבריק, לבן פנינה, וגווני כחול-סגול עקביים בכל הרכיבים.
- הדף הראשי מציג טקסט ברור שהוא דף הבית הראשי תחת ה-Layout החדש.
- כל הדפים העתידיים ירושו את ה-Layout כך שהתוכן יוצג תמיד בין ה-Header ל-Footer.

**הערות:**

- יש להקפיד על סטיילינג עקבי באמצעות Tailwind כך שהמראה יהיה מקצועי ונקי.
- לוגו זמני יכול להיות טקסטואלי עד לחיבור לגרפיקה רשמית.

**סטטוס:** הושלם בגרסה 0.0.2 – Layout פרימיום עם Header/Footer קבועים, טיפוגרפיה אחידה ותמיכה מלאה בכל הדפים.

Task 3:
יצירת auth users RBAC עם Supabase – מוכן לפרודקשן

**הגדרת שלב:**

- הקמת שכבת Authentication מלאה מול Supabase Auth, כולל הרשמה, התחברות, ושחזור.
- הגדרת מודל RBAC מפורט (תפקידי מערכת, הרשאות פעולה בכל מודול).
- אינטגרציה מלאה עם ה־app router כך שכל מסך ידע להגיב להרשאות שהוגדרו.
- חיבור לוגים/אודיט בסיסיים לצורך נראות ואבטחה.

**שאלות פתוחות (נדרשות לפני התחלה):**

1. אילו תפקידי מערכת נדרשים בפרודקשן (למשל: admin, manager, agent, viewer)? האם יש היררכיה נוספת?
2. האם יש סכמה מוגדרת לטבלאות פרופיל משתמש/הרשאות שכבר קיימת ב-Supabase או שיש לעצב אחת חדשה?
3. איזה זרימות אימות חובה (OTP, magic link, סיסמה בלבד) ובאילו שפות תצטרך התמיכה הראשונית?
4. האם נדרש חיבור לשירותי צד־שלישי (כגון SSO עם Google/Azure) כבר בגרסה הזו?
5. מהן הדרישות לאחסון סודות/מפתחות (env naming conventions, הגדרות deployment ל-Vercel)?

**סטטוס:** ממתין להגדרת הדרישות הסופיות והשאלות לעיל.
